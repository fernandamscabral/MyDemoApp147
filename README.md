# Mobile Test Automation – MyDemoApp

This repository contains Android mobile test automation examples for the [Sauce Labs MyDemoApp](https://github.com/saucelabs/my-demo-app-android), using Appium, WebDriverIO, Cucumber, Sauce Labs, and Page Object Model.

The project was developed during the *Formação em Teste de Software e QA* at [Iterasys](https://iterasys.com.br/) (class 147), as part of the mobile testing module.

The main goal of this repository is to compare different approaches to automating the same product selection flow using mobile test automation tools and architectures.

---

## 📚 Context

The training focused on comparing different approaches to mobile test automation rather than expanding test coverage across many application features.

Using the same product selection flow, the repository demonstrates the evolution from standalone Appium scripts to WebDriverIO with Cucumber and finally to a Page Object Model implementation.

It also includes the original WebDriverIO browser example used during the course to introduce mobile browser automation before moving to native Android application testing.

The implementation was developed alongside instructor guidance as part of the training activities.

## 🧠 Testing Approach

This repository contains multiple implementations of the same mobile testing scenario.

The main flow used in the MyDemoApp examples is:

1. Open the MyDemoApp application
2. Select a product from the product list
3. Open the product details screen
4. Validate the product name
5. Validate the product price

The purpose is to show how the same flow can evolve from a simple script to a more structured automation approach.

## 🔐 Sauce Labs Environment Variables

The Sauce Labs examples use environment variables for credentials.

Before running Sauce Labs tests, configure the variables in the terminal:

```bash
export SAUCE_USER=your_username
export SAUCE_KEY=your_access_key
```

To check if they were configured:

```bash
echo $SAUCE_USER
echo $SAUCE_KEY
```

These variables are temporary when configured this way. If the terminal is closed or the computer is restarted, they need to be configured again.

Credentials should not be committed to the repository.

## ▶️ How to Run the Tests

Install dependencies:

```bash
npm install
```

---

### 1. Run the standalone local Appium script

Start Appium in one terminal:

```bash
npx appium
```

Open an Android emulator using Android Studio.

Then run:

```bash
node tests/mobile/exemplo.spec.js
```

---

### 2. Run the standalone Sauce Labs script

Make sure the Sauce Labs environment variables are configured.

Then run:

```bash
node tests/mobile/exemplo_sauce.spec.js
```

---

### 3. Run the WebDriverIO browser example locally

Start Appium in one terminal:

```bash
npx appium
```

Open an Android emulator using Android Studio.

Then run:

```bash
npm run wdio
```

---

### 4. Run MyDemoApp with Cucumber on Sauce Labs

Make sure the Sauce Labs environment variables are configured.

Then run:

```bash
npm run wdio-sauce
```

---

### 5. Run MyDemoApp with Cucumber and Page Objects on Sauce Labs

Make sure the Sauce Labs environment variables are configured.

Then run:

```bash
npm run wdio-sauce-po
```
